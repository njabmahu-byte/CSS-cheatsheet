Properties & Rules

The Box Model — every element is a box
Margin — space outside the border
Border — the visible edge
Padding — space inside the border
Content — your actual text or image
Units — how you measure things in CSS
px
Pixels. Fixed size — always the same regardless of screen or parent.
font-size: 16px; padding: 20px;
%
Percentage of the parent element's size. Good for responsive widths.
width: 100%; margin: 0 auto;
rem
Relative to the root font size (usually 16px). 1rem = 16px, 2rem = 32px.
font-size: 1.5rem; gap: 2rem;
em
Relative to the current element's font size. Can compound — use rem instead.
padding: 1em; /* = 1x current font size */
vw / vh
Viewport width/height. 100vw = full screen width. Good for hero sections.
width: 100vw; min-height: 100vh;
auto
Browser calculates the value. Used for centering and natural sizing.
margin: 0 auto; width: auto;
Color formats
Named colors
Plain English color names. Limited but easy to remember.
color: white; background: black;
Hex
Six-digit code starting with #. The most common format on the web.
color: #3b2a1a; background: #f5f0e8;
RGB
Red, Green, Blue values from 0–255. Easier to read than hex for some.
color: rgb(59, 42, 26);
RGBA
Same as RGB but with an alpha (opacity) value from 0 (invisible) to 1 (solid).
background: rgba(0, 0, 0, 0.5); /* 50% black */
HSL
Hue, Saturation, Lightness. Most intuitive for adjusting shades.
color: hsl(30, 40%, 20%);
Specificity — which CSS rule wins when two conflict
Highest
Inline style
style="color: red" on the HTML tag itself
2nd
ID selector
#hero { color: red; }
3rd
Class selector
.menu-row { color: red; }
4th
Element selector
p { color: red; }
Lowest
Universal selector
* { color: red; }
Override all
!important
color: red !important; — use sparingly
Golden rules
  • 1CSS is read top to bottom. If two rules have equal specificity, the one that appears later in the file wins.
  • 2You link your CSS file in the HTML head: <link rel="stylesheet" href="../css/style.css">. Your pages are in pages/ so you need ../ to go up a level.
  • 3The Box Model is everything. Every element has content, padding, border, and margin. Understanding this solves 90% of spacing issues.
  • 4Use box-sizing: border-box on everything. Add * { box-sizing: border-box; } at the top of every CSS file. Without it, padding and border add to the element's width unexpectedly.
  • 5margin: 0 auto; centers a block element horizontally — but only if the element has a set width.
  • 6Don't use px for font sizes on the body. Use rem or a % so the text respects the user's browser settings.
  • 7Flexbox is your best friend for side-by-side layouts. display: flex on the parent, then control children with justify-content and align-items.
  • 8Class selectors (.menu-row) are reusable — use them for anything that repeats. ID selectors (#hero) are for unique one-off elements.
  • 9Use CSS variables (--variable-name) for your brand colors and fonts. Define them once at the top in :root and reuse everywhere. Changing one variable updates the whole site.
  • 10Use browser DevTools (F12 → Inspector) to debug CSS live. You can click any element and see exactly which styles are applied and from which line of code.
